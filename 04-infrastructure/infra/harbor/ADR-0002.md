# ADR-0002 – Escolha do Container Registry

## Contexto

Para o projeto de portfólio DevOps, precisamos de um container registry corporativo, confiável, com suporte a autenticação, projetos privados e gerenciamento de imagens.

## Decisão

Escolhemos o **Harbor** self-hosted, implantado via Helm no cluster Kubernetes do lab.

## Justificativa

- Open source e amplamente usado em empresas;

- Suporta autenticação, projetos privados e RBAC;

- Integração com Kubernetes via Ingress;

- Permite replicação futura para múltiplos clusters;

- Permite armazenar imagens de diferentes pipelines (CI/CD).

## Consequências

- Necessário cluster Kubernetes funcional;

- PVCs recomendados para persistência;

- Ingress configurado para acesso externo;

- Docker CLI configurado com login no Harbor.
